# Redwood Peak Bot v1.5 — Media Carousel Control

**Date:** 2026-07-11
**Status:** Approved design
**Builds on:** the shared Supabase, the website's public site (anon key), the portal server-read pattern.
**Scope:** Slice 4 of 5 (lookup✓ registries✓ reputation+burn✓ → **carousel** → orders/ledger/budgets). Cross-cutting: bot + website.

## Goal

A High-Command `/carousel add|remove|list` that manages the homepage media carousel (title, body, image), and rewiring the public carousel to render slides from Supabase instead of the hardcoded array — with a graceful fallback to the current slides when the DB is empty.

## Data + storage

- **New table `carousel_slides`**: `id` uuid pk · `title` text · `body` text · `image_url` text · `sort_order` int · `active` boolean default true · `created_at` timestamptz default now().
- **Public-read RLS** (the one intentional exception to the "no anon policies" rule, because this is public marketing content): RLS enabled, plus a policy allowing `anon`/`authenticated` to `SELECT` rows where `active = true`. No anon insert/update/delete — the bot writes via the service key.
- **Image hosting**: a **public Supabase Storage bucket `carousel`** (created once in the Supabase console). The bot uploads the Discord attachment there and stores the resulting public URL.

## Command — `/carousel` (High Command)

- **`add title:<> body:<> image:<attachment>`** — validates the attachment is an image (`content_type` starts with `image/`); downloads it via `fetch(attachment.url)` → `arrayBuffer`; uploads to the `carousel` bucket under a generated filename; inserts a slide with `sort_order = (max existing) + 1`; replies with a confirmation card.
- **`remove title:<>`** — deletes the slide whose title matches (case-insensitive). Errors if none match or more than one matches (use `list` to see exact titles).
- **`list`** — an embed listing current slides (title + a short snippet of body), so HC knows what's live.

"Replace" = `remove` then `add`. No edit or reorder command in this slice.

## Website rewire

- **`components/sections/MediaCarousel.tsx`**: on mount, fetch active slides client-side via the existing anon client (`getSupabase()` from `lib/supabase.ts`), ordered by `sort_order`. Render each slide's `image_url` in place of today's gradient block (an `<img>` with `object-cover`, `max-width:100%`), keeping the title + body. Auto-advance/pause behaviour unchanged.
- **Fallback**: keep the current four slides as a `DEFAULT_SLIDES` constant (redaction bars intact). A pure `pickSlides(dbSlides, fallback)` returns the DB slides if any, else the fallback — so an empty DB (or missing creds) still shows the original carousel.
- **`lib/carousel.ts`** (website): `loadSlides()` (client fetch) + the pure `pickSlides`.

## Components

- `db/schema-v1.5.sql` — `carousel_slides` table + anon-read policy; comment noting the `carousel` bucket must be created in the console.
- `src/db/carousel.ts` (bot) — `uploadCarouselImage(buffer, contentType, ext)` (→ Storage `carousel`, returns public URL), `addSlide(title, body, imageUrl)`, `removeSlideByTitle(title)` (returns count removed), `listSlides()`, `carouselFilename(ext)` (pure-ish name generator).
- `src/commands/carousel.ts` (bot) — the `/carousel` group; registered in `src/commands/index.ts`.
- `src/lib/embeds.ts` — a small `carouselListEmbed(slides)` for `list`, and a confirmation for `add`/`remove` via `line()`.
- `lib/carousel.ts` (website) + `MediaCarousel.tsx` rewire.

## Testing

- **Unit (vitest):** website `pickSlides` (DB non-empty → DB; empty → fallback); the filename generator (shape/uniqueness). Bot embed `carouselListEmbed` via `.toJSON()`.
- **tsc-verified:** `src/db/carousel.ts` (Storage upload + inserts), the `/carousel` command, the website loader + `MediaCarousel` changes (plus a website `next build`).
- **Live smoke test (user):** create the `carousel` bucket, run the SQL, `/carousel add` with an image, see it on the homepage; `/carousel list`, `/carousel remove`.

## Deployment notes (added to DEPLOY.md)

1. In Supabase: run `db/schema-v1.5.sql`, and create a **public** Storage bucket named `carousel`.
2. Bot: `git pull && npm run build && pm2 restart redwood-bot` (no new env).
3. Website: redeploy (no new env — anon key already set). The carousel now reads the DB, falling back to the built-in slides until you add any.

## Command inventory after v1.5

Added: `/carousel`. Website carousel becomes DB-driven with a static fallback.

## Out of scope

Slide reorder/edit commands; redaction bars on DB slides (plain text body; the fallback keeps its bars). Orders/ledger/budgets (slice 5) untouched.
