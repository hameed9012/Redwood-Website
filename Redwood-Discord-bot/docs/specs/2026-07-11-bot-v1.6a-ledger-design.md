# Redwood Peak Bot v1.6a — The Ledger (economy foundation)

**Date:** 2026-07-11
**Status:** Approved design
**Builds on:** shared Supabase, tone-colored embeds.
**Scope:** The **foundational** sub-slice of the economy (slice 5). The economy decomposes into: **Ledger (this)** → Orders pipeline → Inventory books → Budgets → Financial reporting + website ledger page. Four of those depend on the ledger, so it ships first. This spec covers **only the ledger**.

## Goal

A money ledger the whole economy posts to and reads from: inflows/outflows split into white (clean) and black (off-books) books, a command to record entries, and an on-demand summary of balances. Later sub-slices (orders, budgets, inventory, reporting) call the same `addLedgerEntry` / `computeBalances`.

## Data model

**New table `ledger_entries`:**
- `id` uuid pk · `amount` int (whole dollars, always positive) · `direction` text `inflow|outflow` · `book` text `white|black` · `reason` text · `source` text default `manual` (later slices tag `order`/`budget`/`inventory` for reconciliation tracing) · `created_by` text · `created_at` timestamptz default now()

RLS **on**, no anon policies (internal books; a public-safe white view comes with the later reporting slice). Money is **integer dollars** — ERLC money is whole numbers, so no float/rounding concerns.

## Commands — `/ledger` (High Command only; the black book is sensitive)

- **`record amount:<int> direction:<inflow|outflow> book:<white|black> reason:<text>`** — inserts a manual entry (`source = 'manual'`, `created_by = caller`). Validates `amount` is a positive integer.
- **`summary`** — the financial-summary card: white balance, black balance, per-book inflow/outflow totals, and the most recent few entries.

## Pure logic — `src/lib/ledger.ts`

- Types: `Book = 'white' | 'black'`, `Direction = 'inflow' | 'outflow'`, `LedgerEntry = { amount: number; direction: Direction; book: Book; reason: string; source: string; createdAt: string }`.
- `formatMoney(n: number): string` → `$1,250` (via `toLocaleString`).
- `computeBalances(entries: Pick<LedgerEntry, 'amount' | 'direction' | 'book'>[]): { white: BookBalance; black: BookBalance }` where `BookBalance = { inflow: number; outflow: number; balance: number }` and `balance = inflow - outflow`.

## Components

- `db/schema-v1.6a.sql` — `ledger_entries` table + RLS.
- `src/lib/ledger.ts` — types, `formatMoney`, `computeBalances` (pure).
- `src/db/ledger.ts` — `addLedgerEntry(amount, direction, book, reason, source, createdBy)`, `listEntries(limit?)` (newest first).
- `src/commands/ledger.ts` — `/ledger record|summary`; registered in `src/commands/index.ts`.
- `src/lib/embeds.ts` — `ledgerEmbed(balances, recent)` summary card (info tone; white/black balances as fields; recent entries listed).

## Testing

- **Unit (vitest):** `computeBalances` (mixed inflows/outflows across both books → correct per-book inflow/outflow/balance; empty → zeros); `formatMoney` (thousands separators, zero); `ledgerEmbed` via `.toJSON()` (balances rendered, tone).
- **tsc-verified:** `db/ledger.ts`, the `/ledger` command.
- **Live smoke test (user):** `/ledger record` a couple of inflows/outflows across white and black → `/ledger summary` shows the right balances.

## Out of scope (later economy sub-slices)

Orders pipeline, inventory books, budgets, financial reconciliation report, and the website ledger page. No website changes here. No public-safe ledger view yet (comes with reporting). This slice only builds the ledger spine + `addLedgerEntry`/`computeBalances` for the rest to build on.
