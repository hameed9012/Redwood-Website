# Redwood Peak Bot v1.3 — Serial + Vehicle/Plate Registries

**Date:** 2026-07-11
**Status:** Approved design
**Builds on:** v1.1 (identities/shifts), v1.2 (`/lookup`), the embed system.
**Scope:** Slice 2 of 5 (lookup done → **registries** → reputation+burn → carousel → orders/ledger/budgets).

## Goal

Register firearms (by minted serial) and vehicles (by provided plate) to a member's cover identity, mark them clean/flagged, and make them searchable through `/lookup` — so a serial or plate traces back to an identity (and, if that identity later rotates/burns, to a dead cover).

## Data model (new tables — `db/schema-v1.3.sql`)

Both register to the member's **currently-active cover identity** at registration time. Later identity rotation leaves the gear on the old (now retired) cover.

**`firearms`**
- `id` uuid pk · `serial` text unique (bot-minted) · `kind` text (the firearm type) · `identity_id` uuid → identities · `discord_id` text → members · `status` text `clean|flagged` default `clean` · `flag_note` text null · `issued_at` timestamptz default now()

**`vehicles`**
- `id` uuid pk · `plate` text (user-provided; unique) · `description` text · `identity_id` uuid → identities · `discord_id` text → members · `status` text `clean|flagged` default `clean` · `flag_note` text null · `issued_at` timestamptz default now()

RLS **on**, no anon policies (same as every other table — bot service key only).

## Commands (all High Command)

- **`/serial type:<firearm> owner:@member`** — mint a serial via pure `generateSerial()` (format `RW-482910`: a `RW-` company prefix + 6 digits), register it to the owner's active identity, reply with a registration card. Errors if the owner has no active identity.
- **`/vehicle plate:<> description:<> owner:@member`** — register the provided plate + description to the owner's active identity. Errors if the plate is already registered, or the owner has no active identity.
- **`/flag identifier:<serial-or-plate> status:<clean|flagged> [note:]`** — locate the firearm (by serial) or vehicle (by plate), set `status` + `flag_note`. Errors if the identifier isn't on file.

Registrations are read through `/lookup`; no separate view command.

## Extending `/lookup`

- **`gatherLookup`** additionally searches `firearms.serial` and `vehicles.plate` (exact, case-insensitive) for a text query, and loads the owner member + identity for matched registrations.
- **New result kind `registration`** in `buildLookupResult`: when the query exactly matches a serial or plate on file, return a registration view — identifier, kind/description, `status`, `flag_note`, `issued_at`, and the owner. **Redaction:** HC sees the owner (cover legal name + which employee); non-HC sees the registration and its status but the owner is `▓▓▓▓` (a registered item never unmasks the cover→employee link).
- **Precedence:** an exact registry match wins over name/incident matches (looking up a plate returns its registration first).
- **"Not on file" red flag:** a plate/serial with no registration but present in `incident_parties` → the existing `outsider-dossier`, with a `notOnFile: true` marker rendered as *"not on file"*. Nothing anywhere → `not-found`.
- **HC member files** gain a "Registered gear" line listing the member's serials + plates (redacted/absent for non-HC).

## Components

- `db/schema-v1.3.sql` — the two tables.
- `src/lib/serialGen.ts` — `generateSerial(rng?)` (pure).
- `src/db/registry.ts` — `registerFirearm`, `registerVehicle`, `flagItem` (find-by-serial-or-plate + update), `searchFirearms`, `searchVehicles`, `firearmsForMember`, `vehiclesForMember`.
- `src/lib/lookup.ts` — extend `LookupData` with `firearms`/`vehicles`, add the `registration` result kind + `notOnFile` on outsider, extend `buildLookupResult` with redaction. Pure.
- `src/lib/embeds.ts` — extend `lookupEmbed` with the `registration` branch (flagged → warning tone), add "Registered gear" to the member-file branch; a small `registrationEmbed` for the `/serial` and `/vehicle` reply cards.
- `src/db/lookup.ts` — `gatherLookup` also queries the registry tables.
- `src/commands/serial.ts`, `src/commands/vehicle.ts`, `src/commands/flag.ts` — the three commands; register all in `src/commands/index.ts`.

## Testing

- **Unit (vitest):** `generateSerial` (format `RW-\d{6}`); `buildLookupResult` registration cases — exact serial/plate → registration; HC sees owner, non-HC gets `▓▓▓▓`; not-on-file marker on outsider; registry match beats name match. `lookupEmbed`/`registrationEmbed` via `.toJSON()` (status tone, redaction markers).
- **tsc-verified:** `db/registry.ts`, the `gatherLookup` extension, the three commands.
- **Live smoke test (user):** `/serial`, `/vehicle`, `/flag`, then `/lookup` a serial and a plate as HC and non-HC; confirm the flagged tone and owner redaction.
